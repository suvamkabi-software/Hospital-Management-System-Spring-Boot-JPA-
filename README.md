# 🏥 Hospital Management System

> A robust backend system built with **Spring Boot** and **JPA (Java Persistence API)** to efficiently manage core hospital operations — including patients, doctors, and appointments.


## 📖 About the Project

The **Hospital Management System** is a Spring Boot-based RESTful backend application designed to streamline and digitise the administrative and operational workflows of a hospital. It provides a clean, scalable API layer for managing core entities such as patients, doctors, and medical appointments, using JPA for seamless database interaction.

---

## 🛠️ Tech Stack

| Technology | Description |
|---|---|
| **Java** | Primary programming language |
| **Spring Boot** | Application framework for rapid development |
| **Spring Data JPA** | ORM & database abstraction layer |
| **Hibernate** | JPA implementation |
| **Maven** | Build & dependency management |
| **H2 / MySQL** | In-memory (dev) / Relational database (prod) |

---

## ✨ Features

- 👤 **Patient Management** — Register, update, view, and delete patient records
- 🩺 **Doctor Management** — Manage doctor profiles including specialisations
- 📅 **Appointment Scheduling** — Book, update, and cancel appointments between patients and doctors
- 🗄️ **Persistent Storage** — Full CRUD operations backed by JPA entities and repositories
- ⚙️ **RESTful API** — Clean REST endpoints for integration with any frontend or client
- 🔧 **Maven Wrapper** — Includes `mvnw` / `mvnw.cmd` for zero-install builds
